# 🚀 Discord 音樂機器人開發日誌 (2026-04-08)
## 階段：Phase 3 互動與訊息優化擴充

今天我們對機器人的使用者介面 (UI) 有了長足的改進，不僅加入了豐富的按鈕互動，還解決了頻道訊息過於凌亂的問題。

---

### 1. 全新的「至底播放面板」 (Sticky Dashboard) 📌
- **一鍵式操作面板**：每個卡片現在都有 5 個實體按鈕：
  - `⏸️ 暫停/▶️ 繼續`：依據目前狀態動態變更標籤與顏色。
  - `⏭️ 跳過`：立即切換至下一首。
  - `🔀 隨機`：打亂目前的隊列順序。
  - `📻 電台`：開關智慧推薦模式。
  - `📋 清單`：彈出當前播放清單（Ephemeral 隱藏訊息）。
- **訊息至底邏輯**：機器人運作期間，不論發生什麼操作，舊的卡片都會自動刪除，並在最新位置發送新卡片。保證控制台始終在最後一則訊息。
- **自動清理機制**：當機器人因播放結束或閒置而離開頻道時，會自動收回所有播放卡片。

---

### 2. 播放流程與功能增強 🎵
- **優化 `/play` 反饋**：點擊單曲後會先顯示藍色的「已加入清單」卡片，隨後才彈出綠色的核心播放卡片。
- **YouTube 播放清單支援**：完美解析 `list=` 參數，支援快速匯入整包歌單，並附帶過長自動截斷通知。
- **自訂匯入上限**：新增 `/limit <數字>` 指令，讓管理員自由決定單次匯入的歌曲數量（預設 50，最高可調至 1000）。
- **隨機播放指令**：同步推出獨立指令 `/shuffle` 與對應的按鈕邏輯。

---

### 3. 底層穩定性與性能優化 ⚙️
- **歷史紀錄防護**：播放滿 **30 秒** 後才會寫入資料庫，有效過濾短時間跳歌產生的無效歷史數據。
- **資料庫衝突修復**：修正了 Git 追蹤 `.sqlite` 檔案導致的部署衝突問題，現已將資料庫排除在版本控制外。
- **GCP 部署優化**：更新了 `deploy-to-gcp.sh` 部署腳本，並手動修正了 PATH 路徑，解決了 node 執行環境的問題。

---

### 目前已知狀態
- **環境**：Node.js v20+, FFmpeg, SQLite。
- **指令集**：`/play`, `/skip`, `/pause`, `/resume`, `/queue`, `/history`, `/shuffle`, `/limit`, `/nowplaying`, `/radio`。

**開發者筆記**：下一個階段 (Phase 4) 可以考慮加入「個人化歌單」或「音效過濾器 (Filters)」等進階功能。
